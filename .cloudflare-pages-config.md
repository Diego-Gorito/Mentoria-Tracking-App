## Custom domain

Apos criar o projeto Pages:
1. Aba "Custom domains" -> "Set up a custom domain"
2. Digitar: `tracking.escola.click`
3. CF vai configurar o CNAME automaticamente (zona `escola.click` ja esta no CF)

### DNS resultante (CF faz automatico via Pages custom domain)

```
tracking    CNAME    <pages-project>.pages.dev    (proxied)
```

O path `/api/*` e roteado pelo Worker `mentoria-tracking-api` via route config no `wrangler.toml`.

## Worker secrets (apos primeiro deploy do worker)

```bash
cd /Users/gorito/Dev/Mentoria-Tracking-App

# Postgres connection string (KV2 Hostinger)
npx wrangler secret put DATABASE_URL
# exemplo: postgresql://tracking_app_writer:<pass>@69.62.102.49:6543/mentoria-tracking

# JWT signing key (gerar: openssl rand -base64 32)
npx wrangler secret put JWT_SECRET

# pgcrypto master key (mesma usada em core.tenant_credentials)
npx wrangler secret put VAULT_KEY

# Resend API key (magic link emails)
npx wrangler secret put RESEND_API_KEY

# Shared secret pra rota interna /api/internal/creds
npx wrangler secret put INTERNAL_API_KEY
```

## Sequencia de primeiro deploy (ordem importa)

1. `wrangler login` (interativo, Diego roda no terminal)
2. Criar Pages project no dashboard (acima)
3. Configurar DNS custom domain `tracking.escola.click` (CF faz automatico via Pages)
4. `cd /Users/gorito/Dev/Mentoria-Tracking-App && npx wrangler deploy` (publica Worker)
5. Setar Worker secrets via `wrangler secret put` (acima)
6. Confirmar auto-deploy Pages via GitHub Actions (primeiro push pos-setup)
7. Testar: `curl https://tracking.escola.click/api/health`
