## Story

Como **dev backend (Dex)**, eu quero **pipeline de build do plugin híbrido (`gtm4wp` upstream vendored + `mentoria-gtm-bootstrap.php` custom + `mentoria-config.json` per-deploy) produzindo directory ready pra Hostinger MCP upload**, pra que **endpoint `/deploy` (F-S05) tenha plugin files prontos sem hops manuais**.

## Acceptance Criteria

### AC-1 — Diretório plugin estruturado

- **Given** que ADR-0008 §3.2 define estrutura plugin híbrido
- **When** Dex cria `tracking-api/plugins/gtm4wp-mentoria/`
- **Then** contém:
  - `gtm4wp/` — GTM4WP upstream vendored (pinned version, e.g. v1.18 release 2025)
    - Strategy: git submodule OR `scripts/fetch-gtm4wp.sh` que baixa de https://github.com/duracelltomi/gtm4wp/releases/download/v1.18/gtm4wp-1.18.zip + unzip
    - Dex decide method, documenta no README do plugins dir
  - `mentoria-gtm-bootstrap.php` — 1 arquivo custom ~50 LoC PHP
  - `mentoria-config.json.template` — template Jinja-style placeholders `{{container_id}}` etc.

### AC-2 — Build script gera config per-install

- **Given** que cada deploy tem container_id + brand_slug específicos
- **When** Dex cria `tracking-api/scripts/build-plugin.ts`
- **Then** assinatura:
  ```ts
  export async function buildPlugin(input: {
    container_id: string;       // ex "GTM-WVWQVMP"
    brand_slug: BrandSlug;
    plugin_version: string;     // ex "gtm4wp-1.18+bootstrap-v1"
  }): Promise<{ pluginPath: string; cleanup: () => Promise<void> }>;
  ```
  - Copia `plugins/gtm4wp-mentoria/` pra path temp `/tmp/build-<uuid>/`
  - Lê `mentoria-config.json.template`
  - Replace placeholders + write `mentoria-config.json` no temp dir
  - Retorna `pluginPath = /tmp/build-<uuid>/` + cleanup callback que `rm -rf` post-deploy

### AC-3 — Bootstrap PHP ativa GTM4WP + configura

- **Given** que ADR-0008 §3.2 Opção C define semantics
- **When** Dex cria `tracking-api/plugins/gtm4wp-mentoria/mentoria-gtm-bootstrap.php`
- **Then** content:
  ```php
  <?php
  /**
   * Plugin Name: GTM4WP Mentoria Bootstrap
   * Description: Pre-configures GTM4WP with brand-specific GTM container.
   * Version: 1.0.0
   * Depends: GTM4WP
   * License: GPL-2.0+
   */

  if (!defined('ABSPATH')) exit;

  register_activation_hook(__FILE__, function() {
    $config_path = __DIR__ . '/mentoria-config.json';
    if (!file_exists($config_path)) {
      error_log('Mentoria GTM Bootstrap: mentoria-config.json não encontrado');
      return;
    }

    $config = json_decode(file_get_contents($config_path), true);
    if (!$config || !isset($config['container_id'])) {
      error_log('Mentoria GTM Bootstrap: config inválida');
      return;
    }

    // Pre-popula opções GTM4WP
    update_option('gtm4wp-options', [
      'gtm-code' => $config['container_id'],
      'consent-mode-v2' => true,
      'gtm-include' => 'header-footer',
      // ... outros campos GTM4WP relevantes
    ]);

    // Ativa GTM4WP se uploaded mas inactive
    if (!is_plugin_active('gtm4wp/gtm4wp.php') && file_exists(WP_PLUGIN_DIR . '/gtm4wp/gtm4wp.php')) {
      include_once ABSPATH . 'wp-admin/includes/plugin.php';
      activate_plugin('gtm4wp/gtm4wp.php');
    }
  });
  ```

### AC-4 — Plugin metadata header standard WP

- **Given** que WP requires plugin headers válidos
- **When** WP scan `wp-content/plugins/`
- **Then** `mentoria-gtm-bootstrap.php` aparece em WP-admin Plugins page com nome + descrição + version. License: GPL-2.0+ (compatível GTM4WP GPL-2.0).

### AC-5 — Docker image inclui plugin files

- **Given** que `tracking-api` container precisa ter plugin files pra passar pra MCP
- **When** Dex modifica `tracking-api/Dockerfile`
- **Then** `COPY plugins/ /app/plugins/` adicionado. Size delta ≤10MB confirmed via `docker images` after build.

### AC-6 — F-S05 endpoint chama buildPlugin

- **Given** que F-S05 AC-6 worker dispara deploy
- **When** worker entra em `deployJob`
- **Then**:
  1. `const { pluginPath, cleanup } = await buildPlugin({ container_id: installation.gtm_container_id, brand_slug: installation.brand_slug, plugin_version: 'gtm4wp-1.18+bootstrap-v1' })`
  2. `await provider.deployPlugin({ domain, slug: 'gtm4wp-mentoria', pluginPath })`
  3. `await cleanup()` no `finally` (sempre, mesmo se erro)

### AC-7 — Cleanup temp dir

- **Given** que /tmp accumulate sem cleanup
- **When** cleanup callback executa
- **Then** `rm -rf /tmp/build-<uuid>/` remove tudo. Mesmo se deploy falha (try/finally em worker).

### AC-8 — Unit test buildPlugin

- **Given** que function pure cria filesystem state
- **When** Dex roda vitest
- **Then**:
  ```ts
  it('buildPlugin produz directory válido', async () => {
    const { pluginPath, cleanup } = await buildPlugin({
      container_id: 'GTM-WVWQVMP', brand_slug: 'zerohum', plugin_version: 'gtm4wp-1.18+bootstrap-v1',
    });
    expect(fs.existsSync(pluginPath)).toBe(true);
    expect(fs.existsSync(`${pluginPath}/mentoria-gtm-bootstrap.php`)).toBe(true);
    expect(fs.existsSync(`${pluginPath}/mentoria-config.json`)).toBe(true);
    expect(fs.existsSync(`${pluginPath}/gtm4wp/gtm4wp.php`)).toBe(true);

    const config = JSON.parse(fs.readFileSync(`${pluginPath}/mentoria-config.json`, 'utf-8'));
    expect(config.container_id).toBe('GTM-WVWQVMP');
    expect(config.brand_slug).toBe('zerohum');

    await cleanup();
    expect(fs.existsSync(pluginPath)).toBe(false);
  });
  ```

## Tech Notes

**Decisões herdadas:**
- ADR-0008 §3.2 Opção C (fork mínimo embarcado) — Dex implementa literal.
- GTM4WP licença GPL-2.0 — fork OK.
- Pinning v1.18 (latest stable em 25/05/2026 — Dex confirma com `gh release list` ou similar).

**Patterns a usar:**
- `node:fs/promises` API.
- `crypto.randomUUID()` pra dir name.
- Path absolutos via `path.resolve(__dirname, '../plugins/...')`.
- Template engine simples: replace `{{var}}` patterns. NÃO usar Handlebars (overkill). Função `function applyTemplate(tpl, vars) { return tpl.replace(/\{\{(\w+)\}\}/g, (_, k) => vars[k] ?? '') }`.

**LoC estimado:** ~200 (build script 80 + PHP 50 + template 30 + tests 40).

**GTM4WP version pin:** começar com latest stable em 25/05/2026 (provavelmente v1.18). Bump quando upstream lança security patch via F-S15 runbook procedure.

## Edge Cases

1. **GTM4WP upstream removido/renomeado:** fetch script falha. Documentar URL alternativa + checksum verify (sha256).
2. **mentoria-config.json malformado:** PHP error_log + plugin inativo. F-S06 validator detecta drift (dataLayer ausente).
3. **WP-admin opens UI mostra duplicate plugins (gtm4wp e bootstrap):** intencional. Bootstrap depends GTM4WP via header. Ambos ativos = stack funciona.
4. **GTM4WP option keys mudam major bump:** bootstrap quebra silenciosamente (option não aplicada). F-S06 validator detecta (dataLayer presente mas container errado). F-S15 runbook tem procedure bump.
5. **Concurrent buildPlugin (worker thread parallelism):** UUIDs únicos previnem colisão path /tmp.
6. **Disk full /tmp:** falha graciosa, throw error → worker captures.
