## Acceptance Criteria

### AC-1 — runbook-auto-provisioner-gtm.md criado

- **Given** que ops manual recorrente vai acontecer (Diego é único user)
- **When** Diego (ou Claude delegado) cria `docs/runbook-auto-provisioner-gtm.md`
- **Then** contém 5 seções obrigatórias:

  1. **Quick reference** (comandos comuns):
     - Verificar account status: `redis-cli -h redis HGETALL gtm:account:<id>`
     - Listar installations: `redis-cli -h redis SMEMBERS gtm:install:list`
     - Force revalidate: `curl -X POST $API/api/installations/<id>/revalidate -H "Auth..."`
     - Ver audit log: `redis-cli LRANGE gtm:audit:<id> 0 50`
     - Liberar lock travado: `redis-cli DEL gtm:lock:install:<id>`

  2. **Troubleshooting matrix** (sintoma → causa → fix):

     | Sintoma | Causa provável | Fix |
     |---|---|---|
     | Token Hostinger expirado / 401 | Diego revogou ou Hostinger expirou | Reconectar via `/sites/connect` |
     | Site não aparece em `/sites` | Cache 60s | Click refresh OR aguardar |
     | Drift falso-positivo | Plugin cache (LiteSpeed) interfere com regex | Limpar cache WP + revalidar |
     | Install travado `uploading` | Worker crashed, lock prende 60s | `redis-cli DEL gtm:lock:install:<id>` |
     | Modal SSE não atualiza | Proxy buffer | Felix valida Caddy/Traefik buffering off |
     | dataLayer ausente pós-install | GTM4WP não ativou | Manual activate via WP-admin Plugins page |
     | Container ID errado no DOM | Bootstrap PHP option update falhou | Re-deploy ou edit `gtm4wp-options` manual |

  3. **Redis state inspection:**
     - HGETALL account / install
     - LRANGE audit
     - SMEMBERS list keys
     - EXISTS lock keys
     - TTL lock

  4. **Backup/restore mock storage:**
     - Procedure restore do MinIO snapshot F-S08 (cross-link `runbook-ops.md`)

  5. **Migração futura pra DB:**
     - Link ADR-0008a §4 (sem detalhar)

### AC-3 — Manutenção periódica

- **Given** que GTM4WP recebe security patches
- **When** runbook seção "Manutenção"
- **Then** procedures:

  **Bump GTM4WP version:**
  1. Update vendored: `bash tracking-api/scripts/fetch-gtm4wp.sh --version=1.19`
  2. Bump constant `plugin_version` em F-S13 build script
  3. Rebuild Docker image: `docker build -t tracking-api:v<new> tracking-api/`
  4. Smoke retest em 1 site (zerohum recomendado): reinstall
  5. Se OK: deploy KV8 production

  **Rotação libsodium keypair (F-S02):**
  1. Gerar novo par: `pnpm tsx scripts/generate-libsodium-keypair.ts`
  2. Migration script: decrypt todos tokens com old keypair → re-encrypt com new
  3. Flip env vars `STORAGE_ENCRYPTION_PUBLIC_KEY` + `STORAGE_ENCRYPTION_SECRET_KEY`
  4. Smoke: re-conectar 1 account → verificar funciona
  5. Documentar rotation date in CHANGELOG
