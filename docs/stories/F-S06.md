## Story

Como **dev backend (Dex)**, eu quero **função `validate(domain, expectedContainerId)` implementando o validador 2-stage**, pra que **deploys confirmem dataLayer presence + container ID match no DOM** (não só HTTP 200) — detecta drift cedo.

## Acceptance Criteria

### AC-1 — Função validate exportada

- **Given** que F-S05 endpoint `/revalidate` chama essa função
- **When** Dex cria `tracking-api/src/lib/validator.ts`
- **Then** exporta:
  ```ts
  export interface ValidationResult {
    passed: boolean;
    stage: 'head' | 'full';
    details?: {
      containerMatch: boolean;
      expectedMatch: boolean;
      datalayerMatch: boolean;
      expectedContainerId: string;
    };
    reason?: string;
  }

  export async function validate(domain: string, expectedContainerId: string): Promise<ValidationResult>;
  ```

### AC-2 — Stage 1 HEAD fast-path

- **Given** que site offline ou inacessível deve falhar rápido
- **When** stage 1 executa
- **Then** `fetch('https://' + domain + '/', { method: 'HEAD', signal: AbortSignal.timeout(5000) })`:
  - Se response.status NÃO está em [200-299]: retorna `{ passed: false, stage: 'head', reason: 'HTTP ${status}' }` SEM stage 2
  - Se network error / timeout: retorna `{ passed: false, stage: 'head', reason: 'timeout' | 'network_error' }`
  - Se OK (2xx): continua para stage 2

### AC-3 — Stage 2 Full GET + regex 3 checks

- **Given** que stage 1 passou
- **When** stage 2 executa
- **Then**:
  1. `fetch('https://' + domain + '/', { signal: AbortSignal.timeout(5000) })` + `.text()` → `html: string`
  2. Aplica 3 regex no html:
     - `containerMatch`: `/GTM-[A-Z0-9]{6,8}/.test(html)` — qualquer container válido aparece
     - `expectedMatch`: `html.includes(expectedContainerId)` — literal string match (ex `"GTM-WVWQVMP"`)
     - `datalayerMatch`: `/window\.dataLayer\s*=\s*window\.dataLayer\s*\|\|\s*\[\]/.test(html) || /dataLayer\s*=\s*\[/.test(html)` — GTM4WP boilerplate
  3. Retorna `{ passed: containerMatch && expectedMatch && datalayerMatch, stage: 'full', details: { containerMatch, expectedMatch, datalayerMatch, expectedContainerId } }`

### AC-4 — Timeout total ≤10s

- **Given** que UI espera response rápido (não bloqueia >10s)
- **When** validator inicia
- **Then** worst case: 5s HEAD + 5s GET = 10s. Se exceder: `{ passed: false, reason: 'timeout' }`. Uso de `AbortSignal.timeout(5000)` por request.

### AC-5 — Unit tests com mock HTML

- **Given** que MSW ou nock pode mockar fetch
- **When** Dex roda vitest
- **Then** 5 cenários green:
  1. **Happy path**: HEAD 200 + GET com `window.dataLayer = window.dataLayer || []` + `GTM-WVWQVMP` no html → `passed=true`
  2. **dataLayer ausente**: HEAD 200 + GET sem regex datalayer → `passed=false`, `details.datalayerMatch=false`
  3. **Container errado**: HEAD 200 + GET com `GTM-OUTRA1` mas sem `GTM-WVWQVMP` → `passed=false`, `details.expectedMatch=false`, `details.containerMatch=true`
  4. **Site offline**: HEAD 503 → `passed=false`, `stage='head'`, `reason='HTTP 503'`
  5. **Timeout**: HEAD demora >5s → `passed=false`, `reason='timeout'`

### AC-6 — Smoke test prod

- **Given** que Mentoria já tem GTM live (CLAUDE.md "Brand mentoria — GTM-5J587HS3")
- **When** Dex roda `validate('colegiomentoria.com.br', 'GTM-5J587HS3')` localmente
- **Then** retorna `{ passed: true, stage: 'full', details: { containerMatch: true, expectedMatch: true, datalayerMatch: true }}`

## Tech Notes

**Decisões herdadas:**
- ADR-0008 §3.6 pseudo-código copiado literal — pouco margem pra criatividade.
- ADR-0008 §7 Q3 decisão: rejeita HEAD-only (não vê dataLayer), rejeita Playwright (custo MVP). 2-stage é meio termo.
- Playwright = Onda 1.5 Should (testa dataLayer pós-JS execute — útil pra sites que injetam GTM via JS file).

**Patterns a usar:**
- Native `fetch` Node 20+ — sem deps.
- `AbortSignal.timeout(ms)` Node 20+.
- Regex testing — pre-compile pra performance se chamado N vezes:
  ```ts
  const CONTAINER_PATTERN = /GTM-[A-Z0-9]{6,8}/;
  const DATALAYER_PATTERN_1 = /window\.dataLayer\s*=\s*window\.dataLayer\s*\|\|\s*\[\]/;
  const DATALAYER_PATTERN_2 = /dataLayer\s*=\s*\[/;
  ```
- HTML body pode ser grande (~500KB-2MB sites WP). NÃO usar `.json()` ou parser complete; só `.text()` + regex.
- HTTPS only — assume domain tem TLS (Hostinger free Let's Encrypt sempre on per default).

**LoC estimado:** ~120 (validator 50 + types 10 + tests 60).

## Test Plan

### Unit (vitest com MSW)

```ts
import { describe, it, expect } from 'vitest';
import { setupServer } from 'msw/node';
import { http, HttpResponse } from 'msw';
import { validate } from '../validator';

const HTML_OK = `<html><head><script>window.dataLayer = window.dataLayer || []; ...</script>
<!-- GTM container --> <script src="..."></script>GTM-WVWQVMP</head><body>...</body></html>`;

const HTML_NO_DATALAYER = `<html><head>GTM-WVWQVMP</head><body></body></html>`;

const HTML_WRONG_CONTAINER = `<html><head>...dataLayer = [...] GTM-OUTRO1...</head></html>`;

const server = setupServer();
beforeAll(() => server.listen());
afterAll(() => server.close());
afterEach(() => server.resetHandlers());

describe('validate 2-stage', () => {
  it('happy path: dataLayer + container correto', async () => {
    server.use(
      http.head('https://zerohum.com.br/', () => new HttpResponse(null, { status: 200 })),
      http.get('https://zerohum.com.br/', () => HttpResponse.text(HTML_OK))
    );
    const r = await validate('zerohum.com.br', 'GTM-WVWQVMP');
    expect(r.passed).toBe(true);
    expect(r.details?.datalayerMatch).toBe(true);
  });

  it('dataLayer ausente → fail', async () => {
    server.use(
      http.head('*', () => new HttpResponse(null, { status: 200 })),
      http.get('*', () => HttpResponse.text(HTML_NO_DATALAYER))
    );
    const r = await validate('x.com', 'GTM-WVWQVMP');
    expect(r.passed).toBe(false);
    expect(r.details?.datalayerMatch).toBe(false);
  });

  it('container errado → fail', async () => {
    server.use(
      http.head('*', () => new HttpResponse(null, { status: 200 })),
      http.get('*', () => HttpResponse.text(HTML_WRONG_CONTAINER))
    );
    const r = await validate('x.com', 'GTM-WVWQVMP');
    expect(r.passed).toBe(false);
    expect(r.details?.expectedMatch).toBe(false);
  });

  it('site offline HEAD 503 → fail sem stage 2', async () => {
    server.use(http.head('*', () => new HttpResponse(null, { status: 503 })));
    const r = await validate('x.com', 'GTM-WVWQVMP');
    expect(r.passed).toBe(false);
    expect(r.stage).toBe('head');
  });

  it('timeout → fail', async () => {
    server.use(http.head('*', async () => { await new Promise(r => setTimeout(r, 6000)); }));
    const r = await validate('x.com', 'GTM-WVWQVMP');
    expect(r.passed).toBe(false);
    expect(r.reason).toMatch(/timeout/);
  }, 10000);  // vitest timeout 10s
});
```

## Edge Cases

1. **Site retorna 301 redirect:** native fetch segue redirects por default. Validate `colegiomentoria.com.br` pode redirecionar pra `www.` — captura via final URL. OK.
2. **HTML gigante (10MB+):** `await response.text()` carrega tudo em memória. Mitigação MVP: aceitar (sites Mentoria <2MB típico). Onda 1.5 considerar streaming + regex em chunks.
3. **Container ID em comment HTML:** `<!-- GTM-WVWQVMP -->` matches. Não distingue tag-real vs comment. Aceito MVP — comment intencional ainda indica intenção. Onda 1.5 parser AST se necessário.
4. **GTM4WP versão antiga sem `window.dataLayer = ...`:** algumas versões usam só `var dataLayer = dataLayer || [];`. Regex `/dataLayer\s*=\s*\[/` cobre fallback.
5. **Site com cache CDN agressivo (Cloudflare/etc):** revalidação pós-install pode bater versão cacheada SEM novo plugin. Mitigação UX: explicar "aguarde cache TTL" se drift detectado em <1h pós-install. F-S15 runbook cobre.
6. **HTTPS cert inválido:** native fetch rejeita por default. UI mostra error claro.
7. **Domain sem WWW vs com WWW:** validar exato passed domain. Se Diego registrou install em `zerohum.com.br` mas só WWW responde, validate falha — Diego ajusta installation.
