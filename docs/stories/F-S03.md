## Story

Como **dev backend (Dex)**, eu quero **uma interface `IHostingProvider` definida + factory `getProvider()`**, pra que **endpoints de install dependam de abstração (não de Hostinger MCP específico)** — Onda 2 pluga `WPRestAdapter` sem refactor.

## Acceptance Criteria (Given/When/Then)

### AC-1 — Interface IHostingProvider com 4 métodos canônicos

- **Given** que ADR-0008 §3.3 define 4 operações que o provider deve oferecer
- **When** Dex cria `tracking-api/src/lib/providers/IHostingProvider.ts`
- **Then** a interface é exatamente:
  ```ts
  export interface IHostingProvider {
    listSites(): Promise<Site[]>;
    verifyDomain(domain: string): Promise<boolean>;
    deployPlugin(opts: DeployPluginOpts): Promise<DeployResult>;
    pingToken(): Promise<boolean>;
  }
  ```
  Sem métodos extras (mantém superfície enxuta). Adições futuras = breaking change documentado.

### AC-2 — Types Site / DeployPluginOpts / DeployResult definidos

- **Given** que callers (F-S05 endpoints) precisam saber estrutura
- **When** Dex define types adjacentes à interface
- **Then** existem types:
  ```ts
  export interface Site {
    domain: string;                  // "zerohum.com.br"
    wp_version?: string;             // "6.5.3" se detectado, undefined senão
    php_version?: string;            // "8.2"
    ttfb_ms?: number;                // 280 — opcional
    is_wordpress: boolean;           // false se MCP detecta site não-WP
  }

  export interface DeployPluginOpts {
    domain: string;                  // alvo
    slug: string;                    // "gtm4wp-mentoria"
    pluginPath: string;              // path local no container (ex "/app/plugins/gtm4wp-mentoria")
  }

  export interface DeployResult {
    status: 'success' | 'partial' | 'failed';
    summary?: { successful: number; failed: number };  // file counts (MCP retorna)
    uploadDirName?: string;          // "gtm4wp-mentoria-aB3kZ9pQ" (random suffix MCP)
    errorSummary?: string;           // truncated 500 chars
  }
  ```

### AC-3 — Factory `getProvider()` retorna instância por type

- **Given** que Onda 2 vai ter múltiplos providers
- **When** Dex cria `tracking-api/src/lib/providers/index.ts`
- **Then** exporta:
  ```ts
  export type ProviderType = 'hostinger' | 'wp_rest';

  export function getProvider(
    type: ProviderType,
    credentials: { token: string; wpAdminPassword?: string }
  ): IHostingProvider {
    if (type === 'hostinger') return new HostingerAdapter(credentials);
    if (type === 'wp_rest') throw new Error("Provider 'wp_rest' is Onda 2 — not implemented in MVP F");
    throw new Error(`Unknown provider type: ${type}`);
  }
  ```
  Re-export de `IHostingProvider`, types, `HostingerAdapter` pra callers importarem 1 source.

## Tech Notes

**Decisões prévias herdadas:**
- ADR-0008 §3.3 escolhe **ports/adapters** (espelha ADR-0011 ERP Integration Bridge — pattern já familiar no time).
- Diagrama ASCII em ADR-0008 §3.3 mostra topology: Hono routes → IHostingProvider (port) → HostingerAdapter (MVP) | WPRestAdapter (Onda 2 stub).
- Factory NÃO recebe `IGtmStorage` (separation of concerns — storage é injected nos routes).
- Provider NÃO conhece DB / Redis / encryption — recebe `credentials` plaintext (caller decifra antes).

**Patterns a usar:**
- Class-based impl (não factory function) — permite extension futura via subclass se necessário.
- Constructor accepts `credentials` (plain object) — Dex callsite tipicamente: `const adapter = getProvider('hostinger', { token: await sealDecrypt(account.token_encrypted, ...) })`.
- TODO comments com `@todo F-S04` linkando próximo story (rastreabilidade).
- NÃO usar exception classes específicas ainda (`HostingerError`, etc.) — vem em F-S05 com error middleware.

**LoC estimado:** ~120 (interface 50 + factory 25 + adapter stub 20 + tests 25).

## Edge Cases

1. **Credentials missing token:** factory NÃO valida — adapter constructor pode aceitar undefined e falhar quando método chamado (defer error MVP). Decidir em F-S04: aceitar undefined no constructor mas throw em `pingToken()` se token absent.
2. **Type casting `'hostinger' as any`:** TypeScript force string literal narrows — runtime check ainda válido.
3. **Multiple providers no mesmo request:** factory é stateless, podem ser N instâncias coexistindo. Sem singleton.
4. **WPRestAdapter "Onda 2" path acidental:** error message deve ser inequivocal — Quinn QA pode validar em gate.
5. **Subclass nesting:** Dex NÃO deve criar subclasses extends HostingerAdapter no MVP. Manter superfície enxuta.
