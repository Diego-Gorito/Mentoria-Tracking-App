## Story

Como **dev frontend (Dex)**, eu quero **8 novos componentes React implementados** (SiteCard, BrandSelect, InstallProgressModal, InstallSuccessState, InstallFailureState, TokenInput, HostingerHelpAccordion, AuditLogEntry), pra que **as 4 rotas `/sites/*` tenham building blocks reutilizáveis** seguindo design system existente.

## Acceptance Criteria

### AC-1 — SiteCard renderiza site state completo

- **Given** que UX §3 Tela 3 mostra desktop horizontal + mobile vertical stack
- **When** Dex cria `tracking-app/src/components/sites/SiteCard.tsx`
- **Then** props `{ site: EnrichedSite, onInstall, onRevalidate, onReinstall, onViewDetails }` renderiza:
  - Header: domain (truncate se >40 chars) + WP version
  - Metadata: PHP version, TTFB
  - `<BrandSelect />` (props `value=site.brand_slug`)
  - Status pill `<DotPill />` (variants: `installed` verde / `drift_detected` amarelo / `failed` vermelho / `not_installed` cinza)
  - Container ID se installed
  - Actions: "Instalar/Reinstalar/Revalidar" + overflow menu "Ver detalhes"
  - Responsive: desktop horizontal, mobile (≤640px) stack vertical

### AC-2 — BrandSelect dropdown com 4 brands hardcoded

- **Given** que MVP suporta apenas 4 brands (CLAUDE.md tabela)
- **When** Dex cria `tracking-app/src/components/sites/BrandSelect.tsx`
- **Then**:
  - Props `{ value?: BrandSlug, onChange: (slug: BrandSlug) => void, disabled?: boolean }`
  - Options hardcoded: `mentoria`, `mentoria-app`, `zerohum`, `ifrn`
  - Labels PT-BR humanos: "Colégio Mentoria", "Mentoria APP", "Colégio Zerohum", "Blog IFRN"
  - `<select>` nativo (a11y por default) + `<label>` oculto (sr-only) com `htmlFor`
  - `disabled=true` quando site já installed (prevent change pós-deploy MVP)

### AC-3 — InstallProgressModal não-fechável com 4 steps

- **Given** que UX §3 Tela 5 + UX-003 (Esc disabled durante install)
- **When** Dex cria `tracking-app/src/components/sites/InstallProgressModal.tsx`
- **Then**:
  - Props `{ isOpen: boolean, steps: Step[], currentStep: number, onComplete?: () => void, onError?: (err) => void }`
  - 4 steps fixos: "Conectando Hostinger" → "Instalando plugin" → "Validando dataLayer" → "Registrando audit log"
  - Estado por step: `done` (✓ verde) / `in_progress` (⟳ azul animado) / `pending` (◯ cinza) / `failed` (✕ vermelho)
  - Progress bar % calculado por step completion
  - Tempo gasto por step (mostra "3.2s" pós-completion)
  - Esc key disabled via custom hook ou `onClose` no-op durante install
  - `useFocusTrap` aplicado (a11y)
  - Reduce motion respeitado (sem spinner agressivo se `prefers-reduced-motion: reduce`)

### AC-4 — InstallSuccessState com checkmark animado + 4 CTAs

- **Given** que UX §3 Tela 6 (UX-006: 4 CTAs próximos passos)
- **When** Dex cria `tracking-app/src/components/sites/InstallSuccessState.tsx`
- **Then**:
  - Props `{ domain, containerId, brandSlug, durationSec, onAction: (action) => void }`
  - Checkmark SVG animado scale `0 → 1.2 → 1.0` (300ms ease)
  - Headline "Tracking instalado com sucesso!"
  - Subtitle com domain + container_id
  - Checklist 4 itens verdes: "Plugin GTM4WP ativo / dataLayer detectado / Container configurado / Audit log registrado"
  - "Tempo total: {durationSec} segundos"
  - 4 CTAs (button group): "Abrir site ↗" (`target=_blank`), "Ver audit log", "Instalar outro", "Voltar à lista"
  - Sem confetti (UX-008)

### AC-5 — InstallFailureState com erro copiável + sugestões

- **Given** que UX §3 Tela 7 (UX-007: ID erro copiável)
- **When** Dex cria `tracking-app/src/components/sites/InstallFailureState.tsx`
- **Then**:
  - Props `{ domain, errorCode, errorMessage, errorId, suggestions: string[], onRetry, onReconnect, onCancel }`
  - X vermelho icon
  - "Não consegui instalar o tracking" headline
  - Box `<pre>` mono com detalhe técnico (`errorCode + errorMessage + errorId`)
  - Botão "Copiar ID" pra clipboard navigator.clipboard.writeText
  - Lista numerada "O que tentar:" com suggestions
  - 4 CTAs contextuais: "Tentar novamente", "Reconectar Hostinger", "Copiar ID", "Voltar"

### AC-6 — TokenInput password + eye toggle

- **Given** que UX-011 (mitiga shoulder-surfing) + UX §3 Tela 2
- **When** Dex cria `tracking-app/src/components/sites/TokenInput.tsx`
- **Then**:
  - Props `{ value: string, onChange: (v: string) => void, onValidate?: () => void, error?: string }`
  - Input `type='password'` default
  - Botão toggle visibility com Phosphor `<Eye />` / `<EyeSlash />` icons
  - `aria-pressed` no toggle (a11y)
  - `aria-label="Mostrar/ocultar token"`

## Edge Cases

1. **Domain truncate em SiteCard:** CSS `text-overflow: ellipsis` + tooltip mostra full domain on hover.
2. **BrandSelect com `value` desconhecido:** dropdown mostra "—" placeholder.
3. **InstallProgressModal abre + onComplete não chamado (worker travou):** UI mostra warning após 60s "Demorando mais que esperado". Botão emergency close (manual reset).
4. **Reduced motion:** spinner anim → static icon. Detect via `useMediaQuery('(prefers-reduced-motion: reduce)')`.
5. **TokenInput paste com whitespace:** trim automático antes de salvar.
6. **HostingerHelpAccordion mobile:** stack vertical, tap target ≥44px.
