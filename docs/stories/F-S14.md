## Story

Como **PO (Diego/Nova)**, eu quero **smoke test E2E executado nos 4 brands próprios (mentoria, mentoria-app, zerohum, ifrn)** seguindo fluxo completo (conectar Hostinger → listar sites → instalar tracking → validar dataLayer), pra que **a feature seja **provada** antes de marcar MVP-done** (não só "passa testes unitários").

## Acceptance Criteria

### AC-1 — Diego conecta Hostinger via UI

- **Given** que `/sites/connect` está deployed em staging
- **When** Diego abre browser → login Supabase → navega `/sites/connect`
- **Then**:
  - Form aparece per UX §3 Tela 2
  - Diego cola Hostinger API token real
  - Click "Validar e conectar"
  - Sucesso → toast "Conta conectada" + redirect `/sites`
  - Backend: `redis-cli HGET gtm:account:<id> token_encrypted` retorna string base64 (NÃO plaintext)

### AC-2 — UI /sites lista sites Hostinger reais

- **Given** que account conectado
- **When** Diego navega `/sites`
- **Then**:
  - Lista renderiza pelo menos 4 sites Diego (mentoria, mentoria-app, zerohum, ifrn — ou os 4 que existem em Hostinger account dele)
  - Cada card mostra domain + WP version + brand dropdown vazio
  - Status pill "Não instalado" cinza inicial

### AC-3 — Install flow completo nos 4 brands

**Ordem recomendada (segurança crescente, risco decrescente):**

1. **zerohum.colegiomentoria.com.br** (lowest risk — sem prod tracking ativo)
2. **ifrn.colegiomentoria.com.br** (medium risk — blog dedicado, traffic baixo)
3. **mentoria-app.com.br** (medium-high — tem hotmart edge ativo, NÃO tocar)
4. **colegiomentoria.com.br** (highest risk — prod ativo, last to test)

Pra cada brand:

- **Given** que site selecionado + brand dropdown configurado
- **When** Diego clica "Instalar tracking"
- **Then**:
  - Modal confirm aparece (UX §3 Tela 4) com container_id correto + summary
  - Diego confirma
  - InstallProgressModal abre per UX §3 Tela 5
  - 4 steps todos completam status `done` em <2min p95 (PRD §5.1)
  - InstallSuccessState modal aparece (UX §3 Tela 6) com 4 checklist verdes
  - `useAuditLog` mostra entry `installation_completed` no audit log do site

### AC-4 — Validação manual F12 dataLayer no site real

- **Given** que install completou
- **When** Diego abre o site em nova aba (CTA "Abrir site ↗")
- **Then** browser DevTools (F12):
  - Console: `window.dataLayer` retorna array não-vazio
  - Network tab: request `gtm.js` com container_id correto (ex `id=GTM-WVWQVMP`)
  - Cookies: cookies GTM presentes
- Diego confirma manualmente cada brand.

### AC-5 — Tempo p95 <2min

- **Given** que PRD §5.1 define target
- **When** Diego mede tempo total modal (success state mostra "Tempo total: Xs")
- **Then** soma 4 brands → p95 <120s. Documentar tempo de cada em smoke-results.md.

### AC-6 — Drift detect smoke

- **Given** que install OK em 1 brand (sugestão: zerohum)
- **When** Diego desativa plugin manualmente via WP-admin (hPanel → Plugins → Deactivate GTM4WP Mentoria Bootstrap)
- **Then**:
  - Diego volta na UI `/sites/<id>` → click "Revalidar"
  - F-S06 validator detecta dataLayer ausente
  - Status muda pra `drift_detected` amarelo
  - Explicação UI: "dataLayer ausente — plugin desativado?" (UX-014)

### AC-7 — Reinstall smoke

- **Given** que site em drift_detected (AC-6)
- **When** Diego clica "Reinstalar tracking"
- **Then**:
  - Modal confirm aparece
  - Install completa
  - Status volta `installed` verde
  - Audit log mostra ambos entries: `validation_failed` (drift) + `installation_completed` (reinstall)

### AC-8 — Failure smoke (controlado)

- **Given** que install funcional em 1 brand
- **When** Diego revoga token Hostinger via hPanel (gera novo, antigo morre)
- **Then** Diego tenta install em outro site (não tocou):
  - `pingToken()` detecta 401
  - Backend retorna error
  - Modal failure aparece (UX §3 Tela 7) com:
    - errorCode = `INVALID_TOKEN`
    - Sugestão "Reconectar Hostinger"
    - ID erro copiável
  - Diego clica "Reconectar Hostinger" → form `/sites/connect` com novo token → retry install → sucesso

## Edge Cases

1. **1 brand falha install:** NÃO marcar story done. Abrir bug + retornar Sprint 1/2 fix. F-S14 reiniciado após fix.
2. **Tempo >2min em 1 brand:** investigar (rede slow? site pesado? MCP lento?). Documentar nota — não fail se overall p95 OK.
3. **Container Mentoria + IFRN compartilham GTM-5J587HS3:** correto per CLAUDE.md. dataLayer presença + container match em ambos ok mesmo "duplicado".
4. **WP-admin não acessível durante drift smoke:** Diego usa hPanel File Manager pra deletar plugin → mesma deactivation effect.
5. **Multiple tokens Hostinger:** smoke usa apenas 1. Multi-account = Onda 2.
6. **Validação manual F12 (não-automatizada):** aceito MVP. Onda 1.5 Playwright cobre.
