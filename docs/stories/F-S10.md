## Story

Como **dev frontend (Dex)**, eu quero **4 rotas Vite (`/sites`, `/sites/connect`, `/sites/:id`, `/sites/:id/logs`) com pages compostas dos components F-S09**, pra que **Diego navegue pelo flow completo** via menu sidebar.

## Acceptance Criteria

### AC-1 — /sites SitesListPage com empty + lista

- **Given** que UX §3 Tela 1 (empty state) e Tela 3 (lista de cards)
- **When** Dex cria `tracking-app/src/pages/SitesListPage.tsx`
- **Then**:
  - Render condicional: se `useHostingerAccount().isConnected === false` → `<EmptyState />` per UX §3 Tela 1 + CTA "Conectar via Hostinger" (link `/sites/connect`)
  - Se connected → header (filtros + refresh + "Conectar conta" button secundário) + lista `<SiteCard />` map sobre `useSites().sites`
  - Loading state skeleton 3 cards
  - Error state EmptyState variant=error + "Tentar de novo"
  - Header opcional `<KpiCard />` com "N sites conectados / N instalados / N drift"

### AC-2 — /sites/connect ConnectHostingerPage form

- **Given** que UX §3 Tela 2 form completo
- **When** Dex cria `tracking-app/src/pages/ConnectHostingerPage.tsx`
- **Then**:
  - Link "← Voltar" (navigate `-1`)
  - Header "Conectar Hostinger" + subhead
  - `<HostingerHelpAccordion defaultOpen={true} />` (F-S09)
  - Field apelido opcional (input text)
  - `<TokenInput />` (F-S09)
  - CTAs "Validar e conectar" / "Cancelar"
  - Submit chama `useHostingerAccount().connect(token, label)` (F-S11)
  - Success → toast "Conta conectada" + navigate `/sites`
  - Error → toast erro + mantém form
  - **NÃO** modal — rota dedicada (UX-002)

### AC-3 — /sites/:siteId SiteDetailPage com tabs

- **Given** que UX-013 audit log em rota dedicada bookmarkable
- **When** Dex cria `tracking-app/src/pages/SiteDetailPage.tsx`
- **Then**:
  - Tabs: "Overview" / "Audit Log" / "Settings"
  - Overview: metadata + último install + actions (Revalidar / Reinstalar / Desinstalar com `<ConfirmDialog />`)
  - Audit Log: `<AuditLogEntry />` (F-S09) em loop sobre `useAuditLog(siteId).entries`
  - Settings: placeholder "Configurações avançadas — Onda 1.5" (reserved)
  - URL param `?tab=audit` muda tab inicial (deep link)

### AC-4 — /sites/:siteId/logs SiteAuditLogPage standalone

- **Given** que UX-013 audit pode ser bookmarked direto
- **When** Dex cria `tracking-app/src/pages/SiteAuditLogPage.tsx`
- **Then**:
  - Mesma view do tab "Audit Log" mas full-page (não embed dentro de tabs)
  - Header com domain + breadcrumb "Sites / [domain] / Logs"
  - Lista `<AuditLogEntry />` paginada (MVP carrega últimas 50)
  - Filtros opcionais: action type, date range (Onda 1.5 — placeholder)

### AC-5 — Sidebar entrada adicionada

- **Given** que UX §2.1 + UX-001 "Sites Conectados" entre Dashboard e Configurações
- **When** Dex modifica `<Sidebar />` em `tracking-app/src/components/layout/Sidebar.tsx` (ou path equivalente)
- **Then**:
  - Item novo "Sites Conectados" com ícone Phosphor `<GlobeHemisphereWest />`
  - Badge "NEW" (verde primary) primeiros 14 dias — controle via `localStorage.getItem('sites-feature-seen-at')` ou simple date check `< 2026-06-08`
  - `to='/sites'` link
  - Active state quando rota `/sites/*` aberta

### AC-6 — Roteamento React Router v6

- **Given** que `tracking-app` provavelmente já usa React Router
- **When** Dex modifica `tracking-app/src/router.tsx` ou similar
- **Then**:
  - 4 rotas adicionadas com lazy load `React.lazy()`:
    ```tsx
    {
      path: '/sites',
      element: <Suspense fallback={<PageSkeleton />}>
        <SitesListPage />
      </Suspense>,
    },
    // ... outras 3
    ```
  - Suspense fallback consistente
  - 404 fallback retorna NotFoundPage existente

### AC-7 — Auth guard

- **Given** que rotas `/sites/*` requerem Supabase auth
- **When** usuário não autenticado tenta acessar
- **Then**:
  - `<AuthGuard>` wrapper component check `useUser()` hook (Supabase)
  - Se !user → `<Navigate to="/login" replace />`
  - Pós-login, redirect back via `location.state.from`

### AC-8 — Layout AppShell consistente

- **Given** que tracking-app já tem AppShell (sidebar + topbar)
- **When** 4 pages renderizam
- **Then** todas dentro de `<AppShell>` (reuse layout existente). Topbar mostra título da page atual.

## Edge Cases

1. **siteId inválido (UUID malformado):** SiteDetailPage useEffect fetch retorna 404 → "Site não encontrado" empty state + link voltar.
2. **Logout durante navegação:** AuthGuard re-evalua, redirect login.
3. **Network offline:** banner topo "Sem conexão. Mostrando cache." (UX §3 Tela 1).
4. **Mobile 375px sidebar collapse:** sidebar overlay drawer (já existe pattern AppShell).
5. **Badge NEW após 14 dias:** check date, remove badge silently.
6. **Deep link `/sites/:id?tab=audit`:** parse search param, setInitialTab.
